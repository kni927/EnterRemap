# TASK: メニューバー常時表示アイコンへの切り替え（Phase 5）

## 背景
Phase 4で異常終了通知(UserNotifications)を実装したが、
「通知が出た時点でもう見逃している/忘れている」という実害が
複数回発生した。通知という一過性の仕組みではなく、
「今動いているかどうか」を常時目視確認できる形に切り替える。

## 方針
NSStatusItem によるメニューバーアイコン常駐に変更する。
LSUIElement（Dockに出さない）は維持したまま、メニューバーにのみ
最小限のアイコンを出す。

## 要件
1. NSStatusItem を追加し、稼働中/停止中を色や記号で常時表示する
   - 稼働中（イベントタップ有効）: 例) 緑ドット、または専用シンボル
   - タップ無効化中・復帰待ち: 例) 黄/グレーの警告表示
   - 実装はSF Symbolsまたはシンプルな図形で可（凝ったデザインは不要）
2. アイコンクリックでメニューを表示する:
   - 現在の状態表示（稼働中 / 無効化中 など）
   - 「一時停止」「再開」（イベントタップの有効/無効切り替え。
     プロセス自体は終了しない）
   - 「終了」（アプリ終了。killallと同等）
   - バージョン表示程度は任意
3. Phase 4で追加した UserNotifications 関連コード
   （権限要求、tapDisabled通知、SIGTERM/SIGINT/SIGHUP通知、
   `--test-notification` フラグ）を撤去する
4. 既存のIME判定・composing state・許可リストロジックには触れない
5. 一時停止中はEnter/Cmd+Enterともにパススルー（リマップなし）とし、
   誤動作より「何もしない」を優先する

## README整備（合わせて対応）
1. 通知関連の記載（権限要求の案内等）を削除する
2. メニューバーアイコンでの状態確認・一時停止操作を案内するセクションに
   置き換える（`pgrep`/`killall`の説明は残しつつ、通常はメニューバーで
   足りる旨を明記）
3. known-issues.md の通知検証関連の記載は、通知機能撤去に伴い削除する
   （原因がアクセシビリティ権限未登録によりプロセス未起動だった旨は
   docs の調査ノートに残っているため、そちらの記録で十分）
4. README-ja.md / README.md（英語版）両方に反映する

## 非対象
- 公証(notarization)
- 許可リスト、IME判定ロジックの変更
