# TASK: メニュー微調整（Phase 9）

## 背景
v1.5.1リリース後の実機確認で見つかった細かい改善点。

## 要件

### 1. Login Items設定を開く導線を追加
- メニューに「Open Login Items Settings...」のような項目を追加する
- `NSWorkspace.shared.open` で以下のURLスキームを開く:
  `x-apple.systempreferences:com.apple.LoginItems-Settings.extension`
- READMEのログイン項目登録手順（システム設定を手動で辿る説明）を、
  この項目からワンクリックで開ける旨に更新する

### 2. Hideメニュー項目を撤去
- Phase 8で追加した「Hide EnterRemap」(`NSApplication.hide(_:)`,
  Cmd+H)を削除する
- 原因: `.accessory`ポリシー（LSUIElement相当）のアプリではHideの
  概念が曖昧で、常にグレーアウトし操作不能になっていた。実利がないため撤去
- 併せて、意図せず表示されている標準アイコン（Hide項目に自動付与されて
  いたもの）も消える想定

### 3. Add Custom Appにbundle ID確認方法のヒントを追加
- 「Add Custom App...」のダイアログ（NSAlert）に、bundle IDの調べ方を
  ヒントとして表示する
- 例文（サブタイトルやplaceholder、informativeText等、実装しやすい形でよい）:
  ```
  mdls -name kMDItemCFBundleIdentifier /Applications/<AppName>.app
  ```
  または
  ```
  osascript -e 'id of app "<AppName>"'
  ```
- 両方併記でもどちらか一方でもよい（前者はGUIでApp名がそのまま
  ドラッグ&ドロップ等で分かりやすい、後者はDockに出ているアプリ名で
  引ける、という違いがある程度で優劣はない）

## 動作確認
- Login Items設定がワンクリックで開くこと
- Hideメニュー項目が表示されないこと、メニュー全体の見た目に違和感が
  ないこと（前後の項目の区切り線等を確認）
- Add Custom Appダイアログでヒント文言が表示されること

## 非対象
- IME判定・AXRole判定・対象アプリ設定の永続化ロジック
